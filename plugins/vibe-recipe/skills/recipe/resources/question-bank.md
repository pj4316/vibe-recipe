# Recipe 질문 은행

비개발자도 답할 수 있는 말로 묻습니다. 구현에 필요한데 비어 있는 정보만 고르고, 한 번에 3-5개씩 질문합니다.

## 기본 질문

- 이 기능을 가장 먼저 사용할 사람은 누구인가요?
- 사용자는 어디에서 이 기능을 시작하나요?
- 사용자가 어떤 순서로 행동하면 성공인가요?
- 성공하면 화면, 파일, 알림, 저장 데이터 중 무엇이 바뀌나요?
- 이번에는 일부러 만들지 않을 것이 있나요?
- 이 기능이 잘 끝났다고 볼 수 있는 장면은 무엇인가요?

## UI와 화면 상태

- 사용자가 처음 봐야 하는 화면이나 버튼은 무엇인가요?
- 로딩 중에는 무엇을 보여주면 좋을까요?
- 결과가 비어 있을 때는 어떤 안내가 필요할까요?
- 실패하면 사용자가 무엇을 이해하고 다시 시도할 수 있어야 하나요?
- 모바일이나 작은 화면에서 꼭 유지되어야 할 행동이 있나요?

## 데이터와 기록

- 새로 저장되는 정보가 있나요?
- 기존 정보를 수정하거나 삭제하나요?
- 사용자가 되돌릴 수 있어야 하나요?
- 같은 요청을 두 번 하면 어떻게 되어야 하나요?
- 결과를 나중에 다시 볼 수 있어야 하나요?

## 권한과 위험

- 이 기능을 사용할 수 있는 사람과 사용할 수 없는 사람이 있나요?
- 결제, 로그인, 권한, 개인정보, 삭제처럼 조심해야 할 부분이 있나요?
- 실패해도 절대 잃으면 안 되는 정보가 있나요?
- 사람 승인이나 확인 창이 필요한 순간이 있나요?

## Red-team 질문

- 같은 행동을 두 번 하면 결과가 한 번만 생겨야 하나요?
- 중간에 실패하면 사용자가 다시 시도할 수 있어야 하나요?
- 권한 없는 사람이 시도하면 아무것도 바뀌지 않아야 하나요?
- 삭제, 배포, 결제, 권한 변경처럼 되돌리기 어려운 작업은 사람 확인이 필요할까요?
- 너무 큰 입력, 빈 입력, 오래된 화면, 느린 외부 서비스가 있어도 사용자가 안전하게 빠져나올 수 있어야 하나요?
- 이 기능이 실패했을 때 절대 남으면 안 되는 중간 상태가 있나요?

## 외부 연동

- 다른 서비스나 API와 연결되나요?
- 외부 서비스가 느리거나 실패하면 사용자에게 무엇을 보여줘야 하나요?
- 외부 결과를 저장하나요, 매번 새로 가져오나요?

## 검증 질문

- “사용자가 A를 하면 B가 보인다” 형태로 성공 기준을 말하면 무엇인가요?
- 꼭 자동 테스트가 필요한 핵심 흐름은 무엇인가요?
- 자동화가 어렵다면 사람이 어떤 순서로 확인하면 되나요?

## Domain 언어와 결정 기록

- 이 기능에서 새로 등장하는 이름, 역할, 상태가 있나요?
- 방금 말한 용어가 기존 용어집의 표현과 같은 뜻인가요, 다른 뜻인가요?
- 사용자가 보는 이름은 cooking metaphor로 두고, 내부 문서에서만 technical term을 써도 괜찮나요?
- agent가 이 기능을 구현할 때 절대 오해하면 안 되는 규칙은 무엇인가요?
- 나중에 바꾸기 어렵고, 맥락 없이는 이상해 보일 수 있는 결정이 있나요?
- 이 결정에 실제 대안이 있었나요, 아니면 명백한 기본값인가요?

## Form/Input 기능

- 사용자가 어떤 값을 입력하나요?
- 필수 입력과 선택 입력은 무엇인가요?
- 잘못 입력했을 때 어떤 안내가 필요하나요?
- 저장 전에 미리보기나 확인 단계가 필요한가요?
- 입력을 중간에 취소하면 어떻게 되어야 하나요?

## File Upload 기능

- 어떤 파일 형식을 받을까요?
- 파일 크기나 개수 제한이 있나요?
- 업로드 중, 성공, 실패 상태는 각각 어떻게 보여야 하나요?
- 업로드한 파일이나 처리 결과를 나중에 다시 볼 수 있어야 하나요?
- 읽을 수 없는 파일이면 사용자에게 무엇을 안내해야 하나요?

## Dashboard/List 기능

- 사용자가 가장 먼저 확인해야 하는 정보는 무엇인가요?
- 정렬, 검색, 필터가 필요한가요?
- 결과가 없을 때 어떤 안내가 필요하나요?
- 항목을 눌렀을 때 어디로 이동하거나 무엇이 열리나요?
- 새로고침이나 실시간 업데이트가 필요한가요?

## Auth/Permission 기능

- 로그인하지 않은 사용자는 무엇을 볼 수 있나요?
- 어떤 역할이 이 기능을 사용할 수 있나요?
- 권한이 없을 때 어떤 안내가 필요하나요?
- 권한 변경이 즉시 반영되어야 하나요?
- 보안상 기록해야 하는 행동이 있나요?

## Data Edit/Delete 기능

- 수정 또는 삭제 전에 확인 단계가 필요한가요?
- 삭제는 완전 삭제인가요, 보관 처리인가요?
- 사용자가 되돌릴 수 있어야 하나요?
- 같은 데이터를 여러 사용자가 동시에 바꾸면 어떻게 해야 하나요?
- 변경 이력이 필요하나요?

## Notification 기능

- 어떤 사건이 알림을 만들어야 하나요?
- 알림은 어디에 보여야 하나요?
- 사용자가 알림을 끄거나 읽음 처리할 수 있나요?
- 같은 알림이 반복되면 묶어야 하나요?
- 알림 실패가 핵심 기능을 막아야 하나요?

# web-app domain preset

- glossary depth:
  - user workflow, screen/state transition, business object를 함께 설명하는 중간 이상 깊이
  - UI label과 backend/integration term이 다를 때 둘의 관계를 분리해 기록
- role/state style:
  - end user, collaborator, admin/operator role을 구분
  - draft, active, blocked, completed 같은 workflow 상태를 명시
- domain tone:
  - workflow-centered, user-facing, product-language-first
- vocabulary priorities:
  - navigation label, CTA, status badge가 같은 개념을 같은 이름으로 부르게 정리
  - component 이름보다 user-visible capability와 domain object 이름을 우선
- default warning:
  - screen 이름이나 component 이름만으로 domain 용어집을 채우지 않음
  - marketing copy와 operational workflow 용어를 같은 상태 이름으로 섞지 않음
